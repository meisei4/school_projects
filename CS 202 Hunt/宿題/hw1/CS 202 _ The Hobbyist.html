HTML Documents: Marking up content and adding structure

Overview

A local hobbyist group has hired you to create a small website comprised of approximately seven different html documents. The website is meant to provide information about the groups hobby, a hobby that involves the cultivation and promoition of a certain plant. You must choose a single plant type from the following: Grapes, Apples, Peppers, and Tomatoes.

Page Descriptions

You must create several html documents where each of these documents is named and described in the subsections that follow. Ensure that each file is located in a single folder named homework1.

Global Requirements

Each of your documents must contain a header and a footer. Each document will contain the same header and footer.

Header

The header is a major navigation element that must include links that are displayed as

  1. Home (linking to the index.html file)
  2. About (linking to the about.html file)
  3. Contact (linking to the contact.html file)

The header must have a logo. The logo is a small, square image that identifies the hobbyist group. The logo is also a link to the index.html document.

In addition to the logo, the header must include the name of the hobbyist group. Something like The Budding Grapevine or The La Crosse Apple Growers.

Footer

The footer is a major navigation element that must include links that are displayed as

  1. About (linking to the about.html file)
  2. Contact (linking to the contact.html file)

The contact information of hobbyist group must be included. This contact information must include a street address and an email contact. The web page is presumed to be authored by the hobbyist group (most likely authored on their behalf by a software contractor).

The footer must also include a copyright notice.

Requirements and Grading

  1. Email a single zip file of your homework1 folder. Do not email a file in any other format (rar, gz, etc...).
  2. Ensure that each image you use is either a) not protected by copyright or b) you own the copyright.
  3. Your grade will be based, primarily, on the following factors:
    1. The correct use and structure of the elements in the document.
    2. The layout of the documents source code (this is not related to how the document appears in a browser when rendered).
    3. The validity. Check validity with the W3C validator
  4. You will not be graded on the visual appearance of the documents when rendered in a web browser. Do not include any styling directives anywhere in your HTML code.
